# Введение в MCEngine

**MCEngine** — это Fabric-мод, который добавляет в Minecraft полноценный JavaScript-движок на базе GraalVM Polyglot. Он позволяет создавать сложную игровую логику, кастомных мобов с уникальным AI и динамические события без необходимости писать и компилировать Java-код.

---

## Как это работает?

MCEngine использует **GraalVM** для выполнения JavaScript кода непосредственно внутри JVM (Java Virtual Machine). Это обеспечивает:

- **Высокую производительность** — JS-код выполняется так же быстро, как Java
- **Безопасный доступ к API** — скрипты имеют доступ только к разрешенным методам через специальные "мосты" (Bridges)
- **Двустороннюю интеграцию** — Java может вызывать JS-функции, а JS может вызывать Java-методы

### Клиентская и серверная стороны

MCEngine разделяет логику на две независимые стороны:

**Серверная сторона (`code/server/`):**
- Управляет мобами, спавном, AI
- Обрабатывает игровые события (урон, смерть, тики)
- Работает на выделенном сервере и в одиночной игре
- Доступ к API: `MCEngineServer`

**Клиентская сторона (`code/client/`):**
- Отвечает за HUD, частицы, звуки
- Обрабатывает ввод игрока
- Работает только на клиенте
- Доступ к API: `MCEngineClient`

!!! warning "Важно"
    Клиентские и серверные скрипты выполняются в разных контекстах. Серверный код не имеет доступа к клиентскому API и наоборот.

## Основные концепции

### 1. Точка входа (main.js)

Каждый проект должен иметь файл `main.js` в папках `client/` и `server/`. Это первый файл, который выполняется при загрузке проекта.

````javascript
// code/server/main.js
console.log("Серверный скрипт загружен!");
MCEngineServer.onInit(() => {
    console.log("Сервер инициализирован");
});
````


### 2. Хуки и коллбэки

MCEngine предоставляет систему хуков — функций, которые вызываются при определенных событиях.

```javascript
// Регистрируем коллбэк на каждый серверный тик
MCEngineServer.onServerTick(() => {
    // Этот код выполняется 20 раз в секунду
});

// Регистрируем коллбэк на смерть моба
MCEngineServer.modifyAI("minecraft:zombie", {
    onDeath: (mob) => {
        console.log("Зомби умер!");
    }
});
```

### 3. Обертки (Wrappers)
Чтобы безопасно работать с объектами Minecraft из JavaScript, MCEngine использует классы-обертки:
MobWrapper — обертка над Entity/LivingEntity/MobEntity
PlayerWrapper — обертка над ServerPlayerEntity
Эти обертки предоставляют упрощенный API для работы с сущностями:

```javascript
var player = MCEngineServer.getPlayer("Steve");
player.sendMessage("Привет, " + player.getName() + "!");
player.giveItem("minecraft:diamond", 10);
```

### 4. Кастомные мобы
Одна из самых мощных функций — создание кастомных мобов с уникальным поведением:

```javascript
var boss = MCEngineServer.createCustomMob("minecraft:zombie", 0, 100, 0, {
    name: "§c§lБОСС",
    health: 500.0,
    scale: 1.5,
    
    onTick: (mob) => {
        // Логика поведения каждый тик
    },
    
    onDeath: (mob) => {
        console.log("Босс побежден!");
    }
});
```

## Команды

MCEngine добавляет несколько полезных команд:

| Команда                | Описание                                   |
|:-----------------------|:-------------------------------------------|
| `/mcengine reload`     | Перезагружает все скрипты текущего проекта |
| `/mcengine run <code>` | Выполняет произвольный JavaScript код      |

!!! tip "Совет"
    Команда `/mcengine run` полезна для тестирования отдельных функций без необходимости создавать файлы:
    ```javascript
    /mcengine run MCEngineServer.log("Тестовое сообщение")
    ```
