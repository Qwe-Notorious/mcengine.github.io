# Клиентский API (MCEngineClient)

!!! warning "Статус: В активной разработке"
    Клиентский API (`MCEngineClient`) в настоящее время находится на стадии активного проектирования и реализации. Поскольку я работаю над MCEngine в одиночку, я внедряю функции поэтапно. Методы, описанные на этой странице, являются плановыми и могут быть скорректированы до финального релиза.

---

## Обзор

В отличие от серверного API, который я уже реализовал для управления миром и мобами, **Клиентский API** будет отвечать за всё, что происходит исключительно на стороне игрока (клиента). Это необходимо для создания функций, которые сервер не может или не должен обрабатывать, чтобы не создавать лишнюю нагрузку на сеть и не требовать постоянной синхронизации.

### Основные направления, над которыми я работаю:
1. **Кастомный интерфейс (HUD):** Возможность отрисовывать собственные виджеты, полоски ресурсов или уникальные элементы поверх стандартного интерфейса Minecraft.
2. **Визуальные и звуковые эффекты:** Локальный спавн частиц (particles) и воспроизведение звуков только для конкретного игрока.
3. **Обработка ввода (Input):** Регистрация собственных клавиш (keybinds) и обработка событий мыши напрямую из JavaScript.
4. **Клиентский рендеринг:** Динамическое изменение отображения моделей и текстур в реальном времени.

## План будущих обновлений (Roadmap)

Ниже представлен список функций, которые я планирую добавить в `MCEngineClient` в ближайших обновлениях.

### 1. Работа с интерфейсом (HUD)
| Планируемый метод | Описание |
| :---------------- | :------- |
| `registerWidget(id, renderCallback)` | Позволит зарегистрировать кастомный виджет, который будет отрисовываться каждый кадр поверх игрового экрана. |
| `removeWidget(id)` | Позволит удалить зарегистрированный виджет с экрана. |

### 2. Частицы и Звуки
| Планируемый метод | Описание |
| :---------------- | :------- |
| `spawnParticle(particleId, x, y, z, count)` | Создаст визуальный эффект частиц на клиенте (не будет виден другим игрокам, если не синхронизирован сервером). |
| `playSound(soundId, volume, pitch)` | Проиграет звук локально для игрока, игнорируя серверные ограничения дальности. |

### 3. Управление и Ввод
| Планируемый метод | Описание |
| :---------------- | :------- |
| `registerKeyBind(keyName, keyCode, callback)` | Создаст новую клавишу в настройках управления Minecraft и привяжет к ней JS-функцию. |
| `isKeyPressed(keyCode)` | Позволит проверить, удерживается ли указанная клавиша в текущий момент. |

## Концептуальный пример использования

*Обрати внимание: этот код пока не является рабочим. Он демонстрирует предполагаемый синтаксис, над которым я сейчас работаю.*

```javascript
// Регистрация собственной клавиши "Действие мода" (по умолчанию на 'K')
MCEngineClient.registerKeyBind("my_mod_action", "key.keyboard.k", () => {
    
    // 1. Проигрываем звук только для этого игрока
    MCEngineClient.playSound("minecraft:entity.experience_orb.pickup", 1.0, 1.0);
    
    // 2. Создаем эффект частиц вокруг игрока
    var player = MCEngineClient.getPlayer(); // Получаем локального игрока
    MCEngineClient.spawnParticle(
        "minecraft:flame", 
        player.getX(), 
        player.getY() + 1, 
        player.getZ(), 
        20 // Количество частиц
    );
    
    // 3. Выводим сообщение в Action Bar
    MCEngineClient.sendActionBar("§eМагическая способность активирована!");
});
```

## Как ты можешь повлиять на разработку?

Поскольку я развиваю этот проект в соло, ваше мнение и идеи для меня критически важны. Если тебе нужна конкретная функция в клиентском API, которой нет в этом списке:

1. Перейди в раздел [Issues на GitHub](https://github.com/).

3. Опиши, какую задачу ты хочешь решить с помощью этой функции.

Я регулярно проверяю эти запросы и добавляю самые востребованные и интересные идеи в свой приоритетный план разработки.
