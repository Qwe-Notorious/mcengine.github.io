# Обертка игрока (PlayerWrapper)

`PlayerWrapper` — это безопасная обертка над `ServerPlayerEntity`, предназначенная для удобного управления игроками из JavaScript. Класс адаптирован под API Minecraft 1.21.11 и скрывает сложные низкоуровневые вызовы, предоставляя простой и интуитивно понятный интерфейс.

!!! note "Примечание"
    Экземпляр `PlayerWrapper` возвращается методами `MCEngineServer.getPlayer()`, `MCEngineServer.getAllPlayers()` или при поиске мобами через `mob.findNearestPlayer()`.

---

## Базовая информация

### getName()
Возвращает никнейм игрока.

**Возвращает:** `String`.

### getUuid()
Возвращает уникальный идентификатор игрока.

**Возвращает:** `String` (UUID в строковом формате).

### getX(), getY(), getZ()
Возвращают текущие координаты игрока.

**Возвращает:** `Number` (double).

### getHealth(), setHealth(health)
Получает или устанавливает текущее здоровье игрока.

**Параметры `setHealth`:**

| Параметр | Тип    | Описание                |
| :------- | :----- | :---------------------- |
| `health` | Number | Новое значение здоровья |

### getMaxHealth()
Возвращает максимальное здоровье игрока.

**Возвращает:** `Number` (float).

### getFoodLevel(), setFoodLevel(level)
Получает или устанавливает уровень сытости (голода) игрока (от 0 до 20).

**Параметры `setFoodLevel`:**

| Параметр | Тип    | Описание              |
| :------- | :----- | :-------------------- |
| `level`  | Number | Уровень сытости (0-20)|

### getXpLevel(), setXpLevel(level)
Получает или устанавливает текущий уровень опыта игрока.

**Параметры `setXpLevel`:**

| Параметр | Тип    | Описание         |
| :------- | :----- | :--------------- |
| `level`  | Number | Новый уровень XP |

## Сообщения

### sendMessage(message)
Отправляет текстовое сообщение в чат игрока.

**Параметры:**

| Параметр  | Тип    | Описание        |
| :-------- | :----- | :-------------- |
| `message` | String | Текст сообщения |

### sendActionBar(message)
Отправляет сообщение в Action Bar (над хотбаром, исчезает через несколько секунд).

**Параметры:**

| Параметр  | Тип    | Описание        |
| :-------- | :----- | :-------------- |
| `message` | String | Текст сообщения |

### sendColoredMessage(message)
Отправляет сообщение в чат с поддержкой цветовых кодов Minecraft (например, `"§aУспех!"`).

**Параметры:**

| Параметр  | Тип    | Описание                      |
| :-------- | :----- | :---------------------------- |
| `message` | String | Текст с цветовыми кодами (`§`)|

## Инвентарь

### giveItem(itemId, count)
Выдает указанное количество предметов игроку.

!!! info "Особенности реализации"
    В Minecraft 1.21.11 используется метод `insertStack` вместо устаревшего `giveItemStack` для корректного добавления предметов.

**Параметры:**

| Параметр | Тип    | Описание                                  |
| :------- | :----- | :---------------------------------------- |
| `itemId` | String | ID предмета (напр. `"minecraft:diamond"`) |
| `count`  | Number | Количество предметов                      |

**Возвращает:** `boolean` (успешно ли добавлены предметы).

### hasItem(itemId, minCount)
Проверяет, есть ли у игрока минимальное количество указанного предмета (суммирует стаки).

**Параметры:**

| Параметр   | Тип    | Описание                          |
| :--------- | :----- | :-------------------------------- |
| `itemId`   | String | ID предмета                       |
| `minCount` | Number | Минимальное требуемое количество  |

**Возвращает:** `boolean`.

### removeItem(itemId, count)
Забирает указанное количество предметов из инвентаря игрока (начиная с первых слотов).

**Параметры:**

| Параметр | Тип    | Описание                |
| :------- | :----- | :---------------------- |
| `itemId` | String | ID предмета             |
| `count`  | Number | Количество для удаления |

**Возвращает:** `boolean` (успешно ли удалено всё запрошенное количество).

### getItemCount(itemId)
Возвращает общее количество указанного предмета во всех слотах инвентаря игрока.

**Параметры:**

| Параметр | Тип    | Описание    |
| :------- | :----- | :---------- |
| `itemId` | String | ID предмета |

**Возвращает:** `Number` (int).

### clearInventory()
Полностью очищает инвентарь игрока (включая броню и руки).

## Движение и телепортация

### teleport(x, y, z)
Телепортирует игрока на указанные координаты.

!!! info "Особенности реализации"
    В Minecraft 1.21.11 для игроков используется безопасный метод `requestTeleport`, который корректно обрабатывает загрузку чанков и предотвращает ошибки движения.

**Параметры:**

| Параметр  | Тип    | Описание           |
| :-------- | :----- | :----------------- |
| `x, y, z` | Number | Целевые координаты |

### setPosition(x, y, z)
Мгновенно перемещает игрока на указанные координаты без анимации телепортации.

**Параметры:**

| Параметр  | Тип    | Описание           |
| :-------- | :----- | :----------------- |
| `x, y, z` | Number | Целевые координаты |

## Способности и состояние

### setGameMode(mode)
Устанавливает режим игры для игрока.

**Параметры:**

| Параметр | Тип    | Описание                                                                              |
| :------- | :----- | :------------------------------------------------------------------------------------ |
| `mode`   | Number | `0` = Выживание, `1` = Творческий, `2` = Приключения, `3` = Наблюдатель |

### getGameMode()
Возвращает текущий режим игры игрока.

**Возвращает:** `String` (например, `"survival"`, `"creative"`).

### giveXp(amount)
Добавляет указанное количество очков опыта игроку.

!!! info "Особенности реализации"
    В Minecraft 1.21.11 используется метод `addExperience` вместо устаревшего `giveXp`.

**Параметры:**

| Параметр | Тип    | Описание               |
| :------- | :----- | :--------------------- |
| `amount` | Number | Количество очков опыта |

### heal(amount)
Восстанавливает здоровье игрока на указанное значение.

**Параметры:**

| Параметр | Тип    | Описание                          |
| :------- | :----- | :-------------------------------- |
| `amount` | Number | Количество восстанавливаемого здоровья |

### isAlive()
Проверяет, жив ли игрок.

**Возвращает:** `boolean`.

## Точка спавна

### setSpawnPoint(x, y, z)
Устанавливает личную точку возрождения игрока.

!!! info "Особенности реализации"
    В Minecraft 1.21.11 используется новый API: создается объект `SpawnPoint` через `WorldProperties.SpawnPoint.create()` и оборачивается в запись `Respawn`.

**Параметры:**

| Параметр  | Тип    | Описание          |
| :-------- | :----- | :---------------- |
| `x, y, z` | Number | Координаты спавна |

### getSpawnPoint()
Возвращает координаты текущей точки возрождения игрока.

**Возвращает:** `String` в формате `"x,y,z"` или `null`, если точка не установлена.

### clearSpawnPoint()
Сбрасывает личную точку возрождения игрока (возрождение будет происходить на мировом спавне).

## Дополнительные возможности

### killWithReason(reason)
Убивает игрока с указанием конкретной причины смерти (отображается в чате, например: "Игрок сгорел").

!!! info "Особенности реализации"
    Вместо прямого вызова `kill()`, метод использует `player.damage()` с источником урона (`DamageSource`) и значением `Float.MAX_VALUE`, что гарантирует смерть и корректную запись причины.

**Параметры:**

| Параметр | Тип    | Описание (допустимые значения)                                                                 |
| :------- | :----- | :--------------------------------------------------------------------------------------------- |
| `reason` | String | `"generic"`, `"fire"`, `"burn"`, `"lava"`, `"drown"`, `"water"`, `"fall"`, `"magic"`, `"starve"`, `"hunger"`, `"void"`, `"world"` |

### distanceTo(other)
Вычисляет расстояние до другого игрока.

**Параметры:**

| Параметр | Тип           | Описание      |
| :------- | :------------ | :------------ |
| `other`  | PlayerWrapper | Целевой игрок |

**Возвращает:** `Number` (расстояние в блоках). Возвращает `-1`, если `other` равен `null`.

### distanceToMob(mob)
Вычисляет расстояние до моба.

**Параметры:**

| Параметр | Тип        | Описание   |
| :------- | :--------- | :--------- |
| `mob`    | MobWrapper | Целевой моб|

**Возвращает:** `Number` (расстояние в блоках). Возвращает `-1`, если `mob` равен `null`.

## Комплексный пример: Система выполнения квеста

Ниже приведен пример того, как методы `PlayerWrapper` используются вместе для создания логики выдачи награды за квест:

```javascript
function completeQuest(playerName) {
    var player = MCEngineServer.getPlayer(playerName);
    
    if (player == null) {
        console.log("Игрок " + playerName + " не найден (возможно, оффлайн).");
        return;
    }

    // 1. Проверяем, выполнил ли игрок условие (например, есть 10 алмазов)
    if (player.hasItem("minecraft:diamond", 10)) {
        // 2. Забираем алмазы
        player.removeItem("minecraft:diamond", 10);
        
        // 3. Выдаем награду: опыт, предмет и сообщение
        player.giveXp(100);
        player.giveItem("minecraft:netherite_sword", 1);
        player.sendColoredMessage("§a§l[КВЕСТ] Поздравляем! Вы получили Меч Незерита и 100 XP!");
        player.sendActionBar("§6Квест выполнен!");
        
        // 4. Телепортируем игрока на точку сохранения
        player.teleport(0, 100, 0);
        
        // 5. Устанавливаем эту точку как его личный спавн
        player.setSpawnPoint(0, 100, 0);
        
        console.log("Игрок " + playerName + " успешно завершил квест.");
    } else {
        // Если условия не выполнены
        var currentDiamonds = player.getItemCount("minecraft:diamond");
        player.sendMessage("§cВам не хватает алмазов. У вас есть: " + currentDiamonds + " из 10.");
    }
}

// Пример вызова из команды или другого скрипта:
// completeQuest("Steve");
```
